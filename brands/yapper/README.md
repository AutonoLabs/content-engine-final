# Yapper Content System

This directory is the canonical GitHub-backed operating system for Yapper social content: brand rules, verified claims, audience definitions, research, Higgsfield production, calendar planning, media, and publishing records.

## Brand standard

- **Voice:** warm, plain, dignified, operational; never clinical, cutesy, or AI-hype
- **Typography:** Public Sans 400/600/800
- **Palette:** Lavender `#7B61AD`, Lavender 700 `#4E3E71`, Ink Plum `#241C36`, Mint `#98FFD9`, Mint Deep `#3ECF9B`, Lavender Mist `#F4F0FA`
- **Logo:** ten dial dots on rotary geometry with the mint finger stop in the lower-right gap
- **Product position:** familiar phone conversations for residents; useful operational insight for care teams
- **Compliance:** Yapper is a companionship and operational-insight service, not a medical device

The canonical brand pack is `inbox/brand-pack/Yapper-Brand-Pack-v4.pdf`.

## Production rules

1. Every calendar post receives media; ordinary Reddit comments are the only normal exception.
2. Higgsfield is the default media source.
3. The assistant chooses the format, model, hook, narrator, and prompt based on the calendar and recent rotation.
4. Rafael Levine generates the assigned asset in Higgsfield, returns it in Discord, reviews the caption, and approves scheduling.
5. Schedule through Blotato; never intentionally publish immediately unless explicitly requested.
6. Adapt ideas natively. Never copy one caption across every platform.
7. The first visible frame and first line must be understood instantly at mobile size.
8. Use one audience, one primary idea, and one CTA per post.
9. Verify every claim and every media asset before scheduling.
10. Never publish or commit resident-private, facility-private, credential, or authenticated-dashboard information.

## Variety framework

Rotate among cinematic micro-stories, family UGC, operator POV, tutorials, myth/reality, approved-stat visuals, split screens, audio-led hooks, founder build notes, carousels, comment replies, and sanitized walkthroughs.

Portfolio target:

- 60% proven structures
- 30% controlled experiments
- 10% creative wildcards

Do not repeat the same format or hook family twice in succession.
