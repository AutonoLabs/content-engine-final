# AI-Tell Monitor

> Keep the banned AI-writing tells current. Quarterly refresh. The trends shift — what was the tell in 2024 ("delve," "tapestry," em-dash chains) is different from 2026. This skill refreshes the list and integrates it back into `docs/anti-ai-feel.md`.

---

## Why this skill exists

AI writing has a moving target. Every few months, a new pattern emerges:

- **2023 era:** "delve into," "tapestry of," "navigate the landscape"
- **2024 era:** "it's not [X], it's [Y]," em-dash chains, "in today's fast-paced world"
- **2025 era:** "elevate your [X]," "unlock the power of," AI-rhetoric patterns
- **2026 era (current):** new tells emerging (e.g., "in an era of," "robust," "comprehensive solution," formulaic contrastive structures)

Static banned lists go stale. This skill does the work:
1. Detects new AI tells
2. Updates the banned list in `docs/anti-ai-feel.md`
3. Updates per-brand voice profiles via inheritance
4. Logs what was banned and why

---

## When to run this skill

- Quarterly (calendar)
- When you notice an emerging pattern in your own output
- When content engagement drops suddenly (may signal tell re-emergence)
- When a major model update ships (often = new tell pattern)
- When the user reports "this sounds like AI" on something the model just wrote

## Refresh process

### Step 1: Pull current tells (research)

Use search + perplexity to get the latest list. Combine into a single research document.

### Step 2: Categorize

For each candidate tell, classify:
- **Tell strength:** weak / medium / strong (how reliably does it signal AI?)
- **Era:** when did this tell emerge?
- **Recovery:** can it be a real human phrase? (e.g., "it's a process" is overused but human too)
- **Banned-by-default:** is it AI-distinct enough that we should ban it?

### Step 3: Update docs/anti-ai-feel.md

- Add new tells to the banned list
- Note when each was first observed
- Update the "current era" callout at top

### Step 4: Cross-reference voice profiles

For each brand in `brands/`, check if the banned phrase is in their existing specific ban list. If not, propagate.

### Step 5: Test

Generate a sample post for each brand. Verify no banned phrases slip through. If something slips, strengthen the rule.

### Step 6: Log

Append to `docs/ai-tell-history.md`:
```
- 2026-Q3: Added "robust" to banned list (weak-2025 tell that became strong-2026)
- 2026-Q2: Added "in an era of" to banned list
- 2026-Q1: Added "comprehensive solution" to banned list
```

---

## Output

A diff to `docs/anti-ai-feel.md` plus the changelog entry. Voice profiles inherit automatically (no per-brand updates needed unless brand-specific contexts amplify the new tell).

---

## What this skill is NOT

- Not a real-time monitor (run quarterly, not constantly)
- Not exhaustive (catches the top ~20 tells per era, not every micro-pattern)
- Not a substitute for human review (every post is human-approved before publish)
